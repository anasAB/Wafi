### State Ownership (PATTERNS.md §5 Decision Tree)

| State | Owner | Tool |
|---|---|---|
| PowerSync-synced data (products, sales, exchange rates) | Server state | PowerSync reactive queries |
| Current in-progress sale (lines, locked rate) | UI state | `sale.store.ts` (Pinia) |
| Sync status / pending count | UI state | `sync.store.ts` (Pinia) |
| Device identity | UI state | `device.store.ts` (Pinia, stubbed) |
| In-progress sale draft (survives app kill) | Local persistent | Dexie (IndexedDB) |

### Vue 3 MVVM Mapping (PRINCIPLES.md §2)

| MVVM Role | Maps to |
|---|---|
| Model | PowerSync local SQLite + Dexie |
| ViewModel | composables (`useXxx.ts`) |
| View | `<template>` in `.vue` files |

**Rule:** No business logic in `<template>`. Templates bind to composable outputs and emit events only.

## Section 2 — Data Layer

### 2a — Supabase Schema (6 tables)

```sql
CREATE TABLE shops (
  id         UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name       TEXT NOT NULL,
  created_at TIMESTAMPTZ DEFAULT now()
);

CREATE TABLE devices (
  id            UUID PRIMARY KEY,
  shop_id       UUID NOT NULL REFERENCES shops(id),
  device_code   TEXT NOT NULL,  -- temp: T-{4hex}; perm: A/B/C...
  registered_at TIMESTAMPTZ DEFAULT now()
);

CREATE TABLE products (
  id        UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  shop_id   UUID NOT NULL REFERENCES shops(id),
  name_ar   TEXT NOT NULL,
  name_en   TEXT,
  price_usd NUMERIC(10,2) NOT NULL CHECK (price_usd >= 0),
  barcode   TEXT,
  photo_url TEXT,
  is_active BOOLEAN DEFAULT true
);

CREATE TABLE sales (
  id                        UUID PRIMARY KEY,   -- generated locally offline
  shop_id                   UUID NOT NULL REFERENCES shops(id),
  device_id                 UUID NOT NULL REFERENCES devices(id),
  device_sequence           INTEGER NOT NULL,
  display_sale_number       TEXT NOT NULL,
  created_at                TIMESTAMPTZ NOT NULL,
  total_usd                 NUMERIC(10,2) NOT NULL,
  total_syp                 NUMERIC(12,0) NOT NULL,  -- SYP integer, no fractions
  exchange_rate_at_sale     NUMERIC(10,2) NOT NULL CHECK (exchange_rate_at_sale > 0),
  payment_method            TEXT NOT NULL CHECK (payment_method IN ('cash_usd','cash_syp','card')),
  amount_received           NUMERIC(12,2),
  amount_received_currency  TEXT CHECK (amount_received_currency IN ('USD','SYP')),
  change_due                NUMERIC(12,2)
);

CREATE TABLE sale_line_items (
  id             UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  sale_id        UUID NOT NULL REFERENCES sales(id),
  shop_id        UUID NOT NULL REFERENCES shops(id),  -- PowerSync routing
  product_id     UUID NOT NULL REFERENCES products(id),
  quantity       INTEGER NOT NULL CHECK (quantity >= 1),
  unit_price_usd NUMERIC(10,2) NOT NULL CHECK (unit_price_usd >= 0),
  line_total_usd NUMERIC(10,2) NOT NULL
);

CREATE TABLE exchange_rates (
  id        UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  shop_id   UUID NOT NULL REFERENCES shops(id),
  device_id UUID NOT NULL REFERENCES devices(id),
  rate      NUMERIC(10,2) NOT NULL CHECK (rate > 0),
  set_at    TIMESTAMPTZ NOT NULL,
  set_by    TEXT NOT NULL DEFAULT 'owner'
);
```

**Decimal precision decisions:**
- USD values → `NUMERIC(10,2)` — 2 decimal places
- SYP values → `NUMERIC(12,0)` — integer (fractions meaningless at 14,500+ rate)
- Exchange rate → `NUMERIC(10,2)`

**Zero-downtime migration rule (ENFORCEMENT.md §6):** All future changes follow expand → migrate → deploy → contract. Never `ALTER TABLE ADD COLUMN NOT NULL` without DEFAULT on live data.

**PowerSync sync rules** (`powersync.yaml` — deployed to PowerSync service):
```yaml
bucket_definitions:
  shop_data:
    parameters:
      - name: shop_id
        token_parameter: shop_id
    data:
      - SELECT * FROM products WHERE shop_id = bucket.shop_id
      - SELECT * FROM sales WHERE shop_id = bucket.shop_id
      - SELECT * FROM sale_line_items WHERE shop_id = bucket.shop_id
      - SELECT * FROM exchange_rates WHERE shop_id = bucket.shop_id
```

Epic 1 connector returns a hardcoded JWT with `shop_id` claim. When real auth ships, only the connector changes — sync rules unchanged.

### 2c — Dexie Draft Database (local-only, never synced)

Fixes the `locked_exchange_rate` gap identified in spec review — this field is required to restore a draft with the correct locked rate after an app kill.

```typescript
// src/data/dexie/draft.db.ts
import Dexie from 'dexie';

interface SaleDraft {
  id:                    string;   // device_id — one draft per device
  device_id:             string;
  shop_id:               string;
  line_items:            DraftLineItem[];
  locked_exchange_rate:  number;   // captured on first addLine()
  selected_payment_method?: string;
  amount_received_so_far?:  number;
  updated_at:            number;   // Unix ms — for 24h purge
}

interface DraftLineItem {
  product_id:    string;
  name_ar:       string;
  quantity:      number;
  unit_price_usd: number;
}

class DraftDatabase extends Dexie {
  drafts!: Dexie.Table<SaleDraft, string>;
  constructor() {
    super('wafi_drafts');
    this.version(1).stores({ drafts: 'id, device_id, updated_at' });
  }
}

export const draftDb = new DraftDatabase();
```

**`sale.store.ts`** — in-progress sale state

```typescript
  deviceSequence:      number         // persisted to localStorage; increments on each confirm
}
```

**`sync.store.ts`** — sync indicator state
```typescript
interface SyncStore {
  status:       'online' | 'offline' | 'syncing'
  pendingCount: number
  lastSyncedAt: Date | null

}
```

### 3b — Cross-cutting Composables

**`useSaleNumber.ts`** — CD-1
- Input: `deviceCode` (device.store), `deviceSequence` (sale.store)
- Output: `nextNumber()` → `"{code}-{6-digit padded}"`, e.g. `"A-000247"`
- Side effect: increments `sale.store.deviceSequence` + persists to localStorage
- Rule: number is immutable once generated

**`useSaleDraft.ts`** — CD-3
- Watches `sale.store` mutations, writes to Dexie debounced 200ms
- `hasDraft`: boolean (checked on POS screen mount)
- `restoreDraft()`: patches `sale.store.lines` + `lockedExchangeRate` from Dexie; does NOT navigate to payment screen
- `clearDraft()`: deletes Dexie record
- On mount: purges drafts older than 24h

**`useBarcodeScan.ts`** — Story 1.1
- USB: global `keydown` listener; burst ≥30 chars/sec ending in Enter → scanner event; <30 → human typing
- Focus guard: scanner bursts intercepted regardless of focused element
- Camera: wraps `@zxing/browser`; checks `navigator.mediaDevices` availability; exposes `cameraAvailable: boolean`
- Output: `onScan(callback)`, `startCamera()`, `stopCamera()`, `cameraAvailable`

**`usePrinter.ts`** — Story 1.4
- Interface: `IPrinterDriver { print(receipt: ReceiptData): Promise<void> }`
- Epic 1 driver: `SimulatedDriver` — logs ESC/POS commands, resolves after 500ms
- Future drivers: `EpsonDriver`, `StarDriver`, `GenericEscPosDriver` — implement same interface
- Adding a new printer model touches only one new driver file; POS code unchanged

### 3c — Feature Composables

**`useExchangeRate.ts`**
- Reads: latest rate from PowerSync `exchange_rates` table
- Writes: INSERT via PowerSync
- 50% guard: `|newRate - currentRate| / currentRate > 0.5` → requires confirmation
- Does NOT own `lockedExchangeRate` — that lives in `sale.store`
- Emits `rateChanged` event; `useSale` watches it

**`useSale.ts`**
- `addLine(productId)`: on first call, captures `useExchangeRate.currentRate` as `sale.store.lockedExchangeRate`
- `totalSyp`: always `totalUsd × sale.store.lockedExchangeRate` — never live rate
- `hasRateChangeNotice`: true when rate changes while `lines.length > 0`
- Blocks `addLine()` when `useExchangeRate.currentRate === null`

**`usePayment.ts`** — CD-5 state machine
- States: `method-selection` → `amount-entry` → `confirming` → `confirmed`
- `back()`: returns to `method-selection`, clears amount input, sale intact
- `cancel()`: closes modal, sale intact
- SYP total uses `sale.store.lockedExchangeRate` — not live rate
- On `confirm()`:
  1. Write sale + line_items to PowerSync local DB
  2. Call `useSaleNumber.nextNumber()`
  3. Clear `sale.store`
  4. Call `useSaleDraft.clearDraft()`
  5. Emit `confirmed(sale)`

**`useSaleHistory.ts`**
- Queries PowerSync: `SELECT * FROM sales WHERE shop_id = ? ORDER BY created_at DESC LIMIT 7 days`
- `reprint(saleId)`: fetches sale + line_items, builds `ReceiptData`, calls `usePrinter.print()`

**`useSync.ts`**
- Bridges PowerSync status events → `sync.store`
- 24h banner: `Date.now() - sync.store.lastSyncedAt > 86_400_000`

## Section 5 — Error Handling & Edge Cases

### Error Surfaces

| Surface | When | Component |
|---|---|---|
| Toast (auto-dismiss 4s) | Action failed, sale safe, user can retry | `AppToast.vue` |
| Banner (persistent, dismissible) | State user must see before next action | Inline per screen |
| Dialog (blocking) | Irreversible action needs confirmation | `AppDialog.vue` |

**Rule (PRINCIPLES.md SoC):** Error detection in composables (`error: string | null` reactive ref). Error display in templates. No try/catch in components.

### Edge Case Map

| EC | Scenario | Implementation |
|---|---|---|
| 1 | Rate missing on first launch | `useSale.addLine()` blocked; widget yellow; "بيع جديد" disabled |
| 2 | 0-priced item | Allowed; receipt prints "0.00" |
| 3 | Pay tapped twice | `usePayment.isOpen` boolean guard |
| 4 | Printer disconnected mid-print | `PrinterDisconnectedError`; toast; button re-enabled; sale already written |
| 5 | Two offline devices sync | UUID uniqueness guarantees no data loss; device_code prevents display number collision |
| 6 | App killed mid-sale | Dexie draft with `locked_exchange_rate`; recovery banner on resume |
| 7 | Print fails, sale recorded | Toast + reprint from history |
| 8 | Rate changed mid-sale | `lockedExchangeRate` unchanged; notice banner shown in SalePanel |
| 9 | Screen rotation | Pinia + Dexie survive remount; Tailwind reflows layout |
| 10 | IndexedDB quota exceeded | `QuotaExceededError` caught; full-screen error with instructions |
| 11 | Device clock changed | `created_at` = local time; PowerSync server reconciles ordering |
| 12 | 50+ line items | SalePanel `overflow-y-auto`; totals computed across all lines |
| 13 | Arabic + mixed numerals | `unicode-bidi: plaintext` on name elements; no special logic |

### PowerSync Error Handling

```
Sync item failure:
  → sync.store.status = 'offline', errorMessage set
  → SyncIndicator: red dot + error icon
  → Item stays in PowerSync upload queue (auto-retry)
  → User can tap indicator → "مزامنة الآن" manual trigger

24h offline banner:
  → useSync: Date.now() - lastSyncedAt > 86_400_000
  → Non-blocking banner in AppHeader
  → Dismissed on next successful sync
```
